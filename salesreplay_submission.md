# SalesReplay：一线汽车销售的AI对话复盘工具

**2026 德勤数字化精英挑战赛 | Team C：AI重塑销售旅程中的人机协作模式**

---

## 一、我们要解决什么问题

一家中等规模的4S店，每年大约有一万条客户线索进来，最终成交的大概一千多单。剩下八千多个客户流失了。

这些客户为什么没买？问CRM系统，里面的记录大概率写着"客户嫌贵"或"客户再考虑"。但这是销售手填的，大部分情况下是敷衍了事。真实原因可能是：客户老婆不喜欢这个颜色、试驾时后排空间让客户失望、或者销售在关键问题上含糊其辞导致客户失去信任。

这些真实原因藏在销售和客户的每一次对话里——微信聊天、电话录音、到店沟通。但没有人复盘这些对话。

原因很简单：一线销售不会复盘。他们每天要跟十几个客户沟通，下班后没有精力逐字回看自己说了什么。4S店的培训是总部统一下发的标准话术手册，但每个客户情况不一样，背下来的话术到现场用不上。

结果就是：**同样的错误反复犯，同样的客户信号反复漏，但销售自己不知道。**

---

## 二、SalesReplay 做什么

SalesReplay 是一个帮一线汽车销售从自己的对话中学东西的复盘工具。

销售把跟客户的聊天记录贴进来（微信截屏转文字、电话录音转文字均可），工具会像一个干了十五年的老销售一样帮他看这段对话，输出四样东西：

**1. 客户真正在想什么**

客户说"再考虑考虑"，到底是真的在考虑还是在找台阶走？工具从上下文中判断客户的真实意图，并标出表面说辞和真实想法之间的偏差。这是传统CRM的意向评级（1-5分）完全做不到的事。

**2. 你哪里没接住**

客户问了电池衰减的问题，你回了句"网上的信息不一定准确"——这等于没回答，客户的信任在这句话上就掉了。工具会逐句检查对话，找出销售没有正面回应客户关切的地方，每条都附上原文引用。

**3. 真正卡住的是什么**

不是笼统地说"客户在犹豫"，而是具体到：卡在价格上、卡在家里人不同意、还是卡在已经倾向竞品了。不同的障碍对应完全不同的后续策略。

**4. 下次怎么开口**

不是"建议加强沟通"这种废话，而是一句能直接拿去用的话。比如："李姐，上次您提到学区的事还在定，我这边整理了一个L7和L6的对比，万一后面换小车也有个参考。"

## 三、为什么这不是又一个AI话术生成器

市面上的销售AI工具走的是反方向——生成一套标准话术让你照着说。SalesReplay做的是从你自己的实战对话里提取经验。

区别在于：**标准话术是别人嚼过的，你的实战复盘才是自己长出来的能力。**

技术上的区别同样关键。市面上的销售分析工具本质上是做情感打分——把一段对话压缩成"情绪偏负面，意向等级3"。这种粗粒度的分类对一线销售几乎没有可操作性。

SalesReplay 让大语言模型像一个真人一样逐句阅读对话，识别客户话语中的意图偏差（嘴上说的 vs 心里想的）、被忽略的信号、情绪转折点，以及销售在哪句话上犯了错。每一个判断都引用对话原文做证据，不凭空推断。

## 四、业务场景与价值

**场景一：日常自我复盘**

销售每天下班前花五分钟，把当天几段重要对话贴进来跑一遍。不需要等主管点评，不需要背话术手册，直接从自己的实战中积累经验。

**场景二：战败线索重新审视**

一个客户三个月前走了，CRM里写着"嫌贵"。把当时的对话记录丢进SalesReplay，发现真实原因其实是"销售没回答客户关于电池衰减的问题，客户对车的可靠性产生了怀疑"。这个洞察直接决定了重新联系这个客户时该说什么。

**场景三：新人快速成长**

新销售最缺的不是产品知识，是读客户的能力——知道客户什么时候在敷衍你、什么时候是真的在考虑。SalesReplay 把老销售的判断力变成了一个可以反复使用的工具，新人每复盘一次对话，就等于被一个老师傅指点一次。

**量化价值估算**：如果一线销售通过复盘减少关键信号遗漏，假设对成交转化率的提升仅为1-2个百分点（从12%到13-14%），对一家年留资一万条的4S店而言，意味着每年多成交100-200单。按单车毛利3-5万元计算，增量收入在300-1000万量级。

## 五、人机协作边界

SalesReplay 的设计原则是**AI分析，人做决定**。

| AI 负责 | 人（销售）负责 |
|---------|---------------|
| 逐句阅读对话，识别意图偏差 | 决定是否采纳分析结论 |
| 找出未被回应的客户关切 | 决定下次沟通的时机和方式 |
| 判断客户的决策阶段和真实障碍 | 决定是否调整报价或策略 |
| 生成下次开口的话术建议 | 决定是否使用、如何修改 |

工具永远不直接接触客户。它不发消息、不打电话、不自动触发任何外部动作。所有输出都是给销售看的参考，最终行动权完全在人手里。

## 六、技术方案

### 整体架构

```
用户（一线销售）
    │
    ▼
┌─────────────────┐
│  Web 前端        │  React + TypeScript + Vite
│  (单页应用)      │  左栏：贴入对话 / 右栏：复盘结果
└────────┬────────┘
         │ HTTP (Vite Proxy 转发)
         ▼
┌─────────────────┐
│  LLM API        │  MiniMax-M2.7
│  (大语言模型)    │  结构化 JSON 输出
└─────────────────┘
```

### 核心分析维度

工具通过一个精心设计的分析提示词，让大语言模型对每段对话输出以下结构化结果：

- **意图偏差检测**：客户表面说辞 vs 真实想法，以及判断依据
- **决策阶段判断**：随便看看 / 认真在比 / 差一个理由就买 / 倾向不买 / 可以推成交
- **障碍归因**：价格 / 车本身 / 销售个人 / 时机 / 家人 / 竞品
- **遗漏信号**：客户提出但销售未正面回应的关切（逐条列出）
- **销售动作评估**：具体的失误和亮点，附对话原文
- **下一步话术**：可直接使用的开场白和重点话题

### 技术选型

| 层 | 选择 | 理由 |
|----|------|------|
| 前端 | React + TypeScript + Vite | 轻量单页应用，开发快，交互简单 |
| UI组件 | shadcn/ui + Tailwind CSS | 开箱即用，界面干净 |
| LLM | MiniMax-M2.7-highspeed | 中文理解能力强，推理成本低，响应速度快 |
| 部署 | 纯前端 + API代理 | 无需后端服务器，Vite开发代理解决跨域 |

### 为什么不需要后端

SalesReplay 当前阶段是一个纯分析工具：输入对话文本，输出分析结果。不涉及用户系统、数据存储、多人协作。前端直接通过Vite代理调用LLM API，架构最简化，三天内即可完成从零到可用的全流程。

## 八、未来可扩展方向

当前版本是一个最小可用产品，验证"对话级深度复盘"这个核心价值。如果进入决赛，以下方向可以继续推进：

- **话术积累**：销售觉得某次建议好用，可以收藏，逐步建立个人话术库
- **多轮对话追踪**：同一个客户的多次沟通串联分析，观察客户态度随时间的变化
- **团队匿名对比**：同一家店的多个销售匿名对比复盘数据，发现共性问题
- **接入CRM**：分析结果自动写回CRM的客户档案，替代销售手填的意向等级

这些扩展都建立在当前已验证的核心能力之上，不需要重新设计架构。
